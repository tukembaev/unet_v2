# Логика прав доступа для задач

## Реализованная функциональность

### 1. Определение роли пользователя
Система автоматически определяет роль текущего пользователя в задаче:
- **CREATOR** (Создатель) - если `task.creator_id === CURRENT_USER_ID`
- **RESPONSIBLE** (Ответственный) - если роль участника = 'RESPONSIBLE'
- **CO_EXECUTOR** (Соисполнитель) - если роль участника = 'CO_EXECUTOR'
- **OBSERVER** (Наблюдатель) - если роль участника = 'OBSERVER'

### 2. Права на смену статуса задачи

#### Создатель (CREATOR)
- Может устанавливать статусы: **Готово**, **Отмена**
- Видит кнопку со статусом и chevron
- Может открыть dropdown для выбора доступных статусов

#### Ответственный (RESPONSIBLE) и Соисполнитель (CO_EXECUTOR)
- Могут устанавливать статусы: **В работе**, **Контроль**
- Видят кнопку со статусом и chevron
- Могут открыть dropdown для выбора доступных статусов

#### Наблюдатель (OBSERVER)
- Не может менять статус
- Видит только текущий статус без chevron
- Кнопка неактивна (disabled)

### 3. Цвета иконок статусов
Иконки в dropdown используют цвета из канбана:
- **Очередь** (PENDING): `text-gray-600 dark:text-gray-400`
- **В работе** (IN_PROGRESS): `text-blue-600 dark:text-blue-400`
- **Контроль** (REVIEW): `text-orange-600 dark:text-orange-400`
- **Готово** (COMPLETED): `text-green-600 dark:text-green-400`
- **Отмена** (CANCELED): `text-red-600 dark:text-red-400`

### 4. Права на добавление подзадач
Кнопка "Добавить подзадачу" доступна для:
- Создатель (CREATOR)
- Ответственный (RESPONSIBLE)
- Соисполнитель (CO_EXECUTOR)

Для остальных ролей кнопка скрыта.

### 5. Права на добавление документов
Кнопки добавления документов (загрузка файла и прикрепление существующего) доступны для:
- Создатель (CREATOR)
- Ответственный (RESPONSIBLE)
- Соисполнитель (CO_EXECUTOR)

Для остальных ролей кнопки скрыты.

### 6. Права на добавление участников
Кнопка "Добавить участника" доступна для:
- Создатель (CREATOR)
- Ответственный (RESPONSIBLE)

Для остальных ролей кнопка скрыта.

## Технические детали

### Текущий пользователь
В данный момент ID текущего пользователя задан константой:
```typescript
const CURRENT_USER_ID = "69298";
```

В будущем это значение должно получаться из контекста авторизации или auth-провайдера.
