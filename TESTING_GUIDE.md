# Руководство по тестированию создания задач

## Предварительные требования

Убедитесь, что бэкенд запущен на `http://localhost:8080` и доступны следующие endpoints:

1. `GET /api/v1/employees/select` - получение списка пользователей
2. `POST /api/v1/users/sync` - синхронизация пользователей
3. `POST /api/v1/tasks` - создание задачи
4. `POST /api/v1/tasks/{id}/files` - загрузка файлов

## Шаги тестирования

### 1. Открытие диалога создания задачи
- Найдите кнопку "Создать задачу" в интерфейсе
- Нажмите на неё
- Должен открыться диалог с формой

### 2. Заполнение формы

#### Название задачи
- Введите название (минимум 3 символа)
- Попробуйте отметить чекбокс "Важная задача"

#### Описание
- Введите описание задачи (опционально)

#### Файлы
- Перетащите файлы в область загрузки
- Или нажмите для выбора файлов
- Максимум 10 файлов, до 10MB каждый

#### Ответственный (обязательное поле)
- Нажмите на селект "Выбрать ответственного"
- Должен открыться список пользователей с аватарами
- Введите текст для поиска по имени или email
- Выберите пользователя
- Проверьте, что отображается аватар и имя

#### Наблюдатели
- Нажмите на селект "Выбрать наблюдателей"
- Выберите несколько пользователей
- Проверьте отображение badges с аватарами
- Попробуйте удалить выбранного пользователя через крестик

#### Соисполнители
- Аналогично наблюдателям

#### Крайний срок (обязательное поле)
- Нажмите на кнопку выбора даты
- Выберите дату в календаре
- Даты в прошлом должны быть недоступны

### 3. Отправка формы

#### Успешное создание
1. Заполните все обязательные поля
2. Нажмите "Создать"
3. Проверьте в Network DevTools:
   - Запрос на `/users/sync` с массивом пользователей
   - Запрос на `/tasks` с данными задачи
   - Запросы на `/tasks/{id}/files` для каждого файла
4. Диалог должен закрыться
5. Форма должна очиститься

#### Валидация
1. Попробуйте отправить форму без ответственного
   - Должна появиться ошибка "Ответственный обязателен"
2. Попробуйте отправить без даты
   - Должна появиться ошибка "Крайний срок обязателен"
3. Введите название менее 3 символов
   - Должна появиться ошибка валидации

### 4. Отмена
- Заполните форму
- Нажмите "Отмена"
- Диалог должен закрыться
- При повторном открытии форма должна быть пустой

## Возможные проблемы

### Пользователи не загружаются
- Проверьте, что endpoint `/employees/select` возвращает данные
- Проверьте формат данных (должен соответствовать UserListItem)

### Ошибка при создании задачи
- Проверьте консоль браузера
- Проверьте Network tab для деталей ошибки
- Убедитесь, что все endpoints доступны

### Файлы не загружаются
- Проверьте размер файлов (максимум 10MB)
- Проверьте, что endpoint `/tasks/{id}/files` принимает multipart/form-data
